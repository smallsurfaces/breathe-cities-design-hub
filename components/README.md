## Standard prototype chrome — PrototypeHeader

**`src/app/_components/PrototypeHeader.tsx` is the standard chrome bar for every hub prototype build.** Every new concept/build adopts it. It is the single source of the "← Back to hub" affordance, the build name, and the per-build "Updated [date]" stamp.

### The rule for new builds

1. Import it: `import { PrototypeHeader } from "<path>/_components/PrototypeHeader"`.
2. Render it as the **first element** of the page (or the route's `layout.tsx`), passing the build's name:
   ```tsx
   <PrototypeHeader buildName="My New Build" />
   ```
3. **Page content always renders BELOW the bar.** PrototypeHeader renders only the bar — never the page body. Do not float content over it.
4. The date auto-resolves from the route via `_data/build-date.ts` (committed `_data/build-dates.json`, regenerated by `npm run build:dates`). Pass `date="YYYY-MM-DD"` only to override.

### Props

| Prop | Type | Notes |
|---|---|---|
| `buildName` | `string` (required) | The bar title. Truncates with an ellipsis on narrow widths. |
| `date` | `string?` (ISO) | Omit to resolve from the route. |
| `commentSlot` | `ReactNode?` | **Map builds** pass their existing `<AnnotationLayer/>` here (see below). Non-map builds omit it. |
| `controls` | `ReactNode?` | Reserved future LEFT-slot build controls. Empty today. |

### Map builds — pass AnnotationLayer as `commentSlot`

Map prototypes wrap the page in a flex column so the bar is not overlapped by the full-bleed map:

```tsx
<div style={{ display: 'flex', flexDirection: 'column', height: '100dvh', overflow: 'hidden' }}>
  <PrototypeHeader buildName="…" commentSlot={<AnnotationLayer … />} />
  <div style={{ flex: 1, position: 'relative', overflow: 'hidden' }}>
    {/* map + fixed overlays */}
  </div>
</div>
```

- **Move** the existing `<AnnotationLayer/>` out of the map container and pass it as `commentSlot` with **identical props** (`storageKey` / `mapAdapter` / callbacks, or `mapRef` / `popupRef` / `onClearSensor`) so saved comments + freeze behaviour are unchanged. Its toggle is `position:fixed`, so it still sits top-right.
- The map container becomes `flex: 1` (replacing `100vh`).
- Any fixed overlay anchored near `top: 0/1rem` must be nudged down by the bar height (~56px) so it does not tuck under the bar.

### Non-map builds — comments come later

Non-map builds omit `commentSlot`; the bar shows a disabled **"Comments — soon"** affordance. Phase-2 comments are not built yet.

### BcChrome builds (roadmap, cities)

The standard bar goes **above** the BC-site recreation. Render `<PrototypeHeader/>` in the route's `layout.tsx` before `<BcHeader/>`; BcChrome (header/footer + its honest "mock" marker) stays as the build's content.

### Notes

- Tokens: shadcn-style semantic aliases only (`bg-background`, `text-foreground`, `border-border`, `text-muted-foreground`, `bg-muted`) — all aliased to `--bc-*` in `globals.css`. No hardcoded hex. Light mode. No emoji.
- The hub home `/` does **not** carry the bar (it is the back-to-hub destination).
- PrototypeHeader replaced the global fixed `HomeNav` (retired from `layout.tsx`) — it now owns back-to-hub on every retrofitted route.

## Navigation

## Buttons

All buttons use `border-radius: var(--bc-border-radius-pill)` (9999px) or `var(--bc-border-radius-md)` (10px).

| Variant | Class | Background | Text | Hover |
|---|---|---|---|---|
| Primary | `button--brand` | `#0071c7` | white | `#23bced` |
| Accent | `button--accent` | `#003574` | white | `#23bced` |
| Outline | `button--outline` | transparent | `#003574` | `#23bced` |
| Outline White | `button--outlineWhite` | transparent | white | `#23bced` |
| Arrow | `button--arrow` | brand/accent | white | lighter | 55×55px circle |
| Link | `button--link` | none | brand | hover colour |
| External | `button--external` | — | brand | — | with external icon |
| Card | `button--card` | — | — | — | overlay on card hover |
| Icon | `button--icon` | — | — | — | icon-only, 55×55px |

Transition: `var(--bc-transition-fast)` (200ms ease-in-out)

## Hero / BlockHeader

- Full-width section
- Large heading: `font-titleLarge` (4rem → 5.25rem)
- Subheading: `font-titleSub` (1.125rem → 1.625rem)
- Background: white or `darkBlue` depending on variant
- Can include contact form (`BlockHeroContactFormH`)

## Cards

- Border-radius: `var(--bc-border-radius-md)` / `10px`
- Shadow: none (flat design)
- Image: `object-fit: cover`, aspect ratios vary (3:2, 4:3, 16:9)
- Hover: `button--card` overlay
